Overview

Forensics + ML

SCADA environments control power, water, and manufacturing. As they connect to IT networks, their attack surface widens. This project builds an intrusion detection stack that ingests both network traces and memory images, extracts forensic artifacts, and feeds them into supervised ML models to distinguish benign operations from targeted attacks.

Methodology

Scenario → Evidence → Model

Attack Simulation & Data Capture

  • Virtualized SCADA water treatment setup with HMI, PLC, and engineering workstation.
  • Simulated DoS, ARP spoofing, and malware-style persistence on control nodes.
  • Captured synchronized PCAP traces and full memory dumps for idle vs. attack states.

Forensic Feature Engineering

  • Used FTK Imager for memory acquisition and triage.
  • Applied Volatility to extract processes, DLLs, network sockets, kernel hooks.
  • Parsed PCAP with Wireshark/Tshark to flag IP/MAC inconsistencies and anomalous flows.
  • Unified artifacts into a structured feature matrix for supervised classifiers.

Highlights

Detection capability across network and memory layers.

Intrusion Signals

  • Detected ARP spoofing via inconsistent MAC–IP mappings in PCAP logs.
  • Flagged suspicious commands and tooling (e.g. netstat, taskkill) in memory images.
  • Correlated abnormal process trees, DLL injections, and rogue connections with attack timelines.
  • Centralized forensic artifacts into a single dataset for repeatable training and audits.

Model Performance

  • Memory-based feature set with KNN achieved 99.98% malware classification accuracy.
  • Evaluated using accuracy, precision, recall, F1-score, and confusion matrices.
  • Demonstrated the value of combining network traces with deep memory forensics.
  • Pipeline design is extensible to real SCADA logs and SIEM integration.
