Intrusion Detection System for SCADA

Forensics and Machine Learning in Cybersecurity

Overview

SCADA systems (Supervisory Control and Data Acquisition) are the backbone of industrial infrastructure, including utilities like power grids, water treatment, and manufacturing lines. As these systems grow increasingly connected to corporate networks and the internet, their susceptibility to cyberattacks also rises.

Our final-year project aimed to develop a comprehensive intrusion detection pipeline combining digital forensics with supervised machine learning. The system successfully detected anomalies from both network and memory data, allowing for real-time or post-incident forensic analysis.

Methodology

  • Simulated attacks (DoS, ARP spoofing, malware injection) on a virtualized SCADA environment.
  • Collected PCAP data and memory dumps during both idle and attack scenarios for ground truth comparison.
  • Used FTK Imager and Volatility to extract registry keys, active connections, DLLs, processes, and kernel hooks.
  • Mapped extracted features to training datasets and applied various classifiers for anomaly detection.
  • Evaluated models using accuracy, precision, recall, F1-score, and confusion matrices.

Key Technologies

  • Wireshark, Tshark: Packet capture and network anomaly detection
  • FTK Imager: Forensic-grade memory acquisition tool
  • Volatility Framework: Memory analysis and malware signature extraction
  • Scikit-learn (Python): Implemented ML algorithms with cross-validation
  • Jupyter Notebooks: Exploratory data analysis and visualization

Highlights

  • Simulated and recorded attacks on a water treatment control system to test the pipeline.
  • Identified suspicious command executions (e.g., netstat, taskkill) in memory dumps.
  • Detected ARP spoofing by analyzing MAC-to-IP inconsistencies in PCAP logs.
  • Integrated forensic logs into a unified CSV file for model ingestion.
  • Achieved malware classification accuracy of 99.98% using KNN with memory-based features.
